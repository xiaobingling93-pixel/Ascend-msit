# 服务化性能数据比对工具
## 概述 (Overview)
### 工具定位
llama 等模型部署时，客户经常会对服务化的性能有要求，人工调优耗时耗力，并且长时间占用客户机器。服务化参数众多，不同参数组合对性能影响较大，且无显然的线性规律.
服务化自动寻优工具支持对mindie进行自动寻优，获取符合要求的最佳参数组合

### 关键特性
- 自动
- 寻优过程不依赖于npu推理

## 快速入门 (Quick Start)
### 安装工具
cd msit
pip install -e .[speed] # 安装相关加速包，提高训练效率。
如果安装失败Could not fetch URL https://mirrors.aliyun.com/pypi/simple/poetry/，可尝试。
pip install --trusted-host mirrors.aliyun.com .[speed]
上述如果三方宝安装失败，可尝试安装较少依赖的三方包，但训练模型时，大数据量时性能较低。
pip install -e . # 安装基本依赖。

### 命令行基础用法

开启profiling实际跑一遍测试脚本，将采集的profiling数据进行解析然后用于训练模型
采集时需开启环境变量export MODEL_EVAL_STATE_COLLECT=True
```
cd msit/msserviceprofiler/modelevalstate
msServiceProfiler train
```
|参数|说明|是否必选|
|---|---|---|
|-i, input_path|输入数据目录，这里所需的数据即profiling的输出路径|是|
|-o,output-path|输出目录，建议输出在modelevalstate下面创建一个/result/lantency目录来存放|否|
```

msServiceProfiler optimizer
寻优时需开启环境变量export MODEL_EVAL_STATE_ALL=True
若采用不依赖profiling的方式进行寻优需开启环境变量export MODEL_EVAL_STATE_IS_SLEEP_FLAG=true
```
|参数|说明|是否必选|
|---|---|---|
|-lb|控制是否从断点恢复寻优过程，默认关闭，用法例如--load_breakpoint 或 -lb|否|
|-d|设置部署策略，即单机或多机部署，默认为单机部署，用法例如--deploy_policy multiple 或 -d multiple|否|
|--backup|决定是否在寻优过程中备份数据，默认False,不开启，用法例如--backup|否|
