# 一键量化配置协议说明

## 概述

### 配置协议简介

一键量化配置协议采用分层结构设计，顶层包含三个核心字段。

- **apiversion**: 用于选择后端量化服务的版本。
- **metadata**: 用于最佳实践匹配的标签。
- **spec**: 具体的量化服务配置字段。

### 两种使用方式

一键量化功能提供了两种重要的使用方式。

#### 指定量化类型匹配最佳实践库

用户可以通过指定 `quant_type` 参数，让工具自动在最佳实践库中匹配最适合的量化配置。

#### 指定配置文件快速启动量化过程

用户可以通过指定 `config_path` 参数，直接使用自定义的配置文件进行量化。

## 一键量化基础配置协议

### 基础配置

#### apiversion - 后端服务版本选择

**作用**: 用于选择后端量化服务的版本，不同的量化服务有着不同的具体配置协议。

**特点**:

- 当前支持的版本：`"modelslim_v0"`、`"modelslim_v1"`。
- 工具根据此字段选择对应的量化服务后端。
- 不同版本的量化服务可能有不同的配置字段和参数要求。

**使用场景**:

- 当指定 `quant_type` 时：工具根据apiversion选择对应的后端服务进行配置匹配。
- 当指定 `config_path` 时：工具根据apiversion选择对应的后端服务解析配置文件。

**示例**:

```yaml
apiversion: "modelslim_v1"  # 使用modelslim_v1版本的量化服务
```

#### metadata - 最佳实践匹配标签

**作用**: 用于描述这份量化配置的标签，用于进行最佳实践匹配。**注意：在指定配置文件时，该字段无效。**

**核心功能**:

1. **配置标识**: 通过 `config_id` 唯一标识配置。
2. **质量评估**: 通过 `score` 评估配置质量，用于排序选择。
3. **特征标签**: 通过 `label` 描述配置特征，用于匹配和过滤。
4. **兼容性验证**: 通过 `verified_model_types` 验证模型兼容性。

**使用场景**:

- **指定quant_type时**: 工具根据model_type和quant_type在最佳实践库中搜索匹配的配置。
- **指定config_path时**: 该字段被忽略，直接使用配置文件中的spec字段。

**字段详解**:

| 字段名 | 作用 | 说明 | 示例 |
|--------|------|------|------|
| config_id | 配置的唯一标识符 | 命名规范：`{模型名称}-{量化类型}` | `"Qwen2.5-7B-Instruct-W8A8"` |
| score | 配置质量评分 | 范围：0.0 - 1.0，用于排序和选择最优配置 | `0.95` |
| label | 配置标签 | 包含量化位数、是否稀疏、是否包含KV缓存等特征信息 | `{w_bit: 8, a_bit: 8, is_sparse: false}` |
| verified_model_types | 已验证支持的模型类型列表 | 确保配置与特定模型类型的兼容性 | `["Qwen2.5-7B-Instruct"]` |

#### spec - 量化服务配置

**作用**: 具体的量化服务配置字段，包含量化策略、处理流程和保存方式等所有具体参数。

**核心功能**:

1. **流水线定义**: 指定量化处理的流水线类型。
2. **处理器配置**: 定义各种量化处理器的参数。
3. **保存策略**: 指定量化结果的保存方式和格式。
4. **数据集配置**: 指定校准数据集。

**使用场景**:

- **指定quant_type时**: 工具从最佳实践库中匹配到的配置的spec字段。
- **指定config_path时**: 用户自定义配置文件中的spec字段，直接用于量化服务。

### 基础配置协议示例

```yaml
apiversion: "modelslim_v1"
metadata:
  config_id: "Qwen2.5-7B-Instruct-W8A8"
  score: 0.95
  label:
    w_bit: 8
    a_bit: 8
    is_sparse: false
    kv_cache: false
  verified_model_types:
    - "Qwen2.5-7B-Instruct"
spec:
  pipeline: "auto"
  process: [ ]
  save: [ ]
  dataset: "mix_calib.jsonl"
```

该配置文件描述了使用V1量化服务进行空处理。

## 量化服务配置协议表

### 量化服务概览

一键量化支持多种量化服务后端，每种服务都有其特定的配置协议和功能特性。

### 量化服务配置协议表

| 协议版本         | 维护策略 | 状态  |
|--------------|------|-----|
| modelslim_v0 | 即将废弃 | 不推荐 |
| modelslim_v1 | 逐步完善 | 推荐  |

#### modelslim_v0 服务配置协议

modelslim_v0量化服务主要由Calibrator、AntiOutlier等旧版接口组成，其配置协议（YAML）基本与原有的Python-API接口保持一致，便于从旧版本平滑迁移。

**相关文档**:

- [Calibrator.md](..%2FPython-API%E6%8E%A5%E5%8F%A3%E8%AF%B4%E6%98%8E%2F%E5%A4%A7%E6%A8%A1%E5%9E%8B%E5%8E%8B%E7%BC%A9%E6%8E%A5%E5%8F%A3%2F%E5%A4%A7%E6%A8%A1%E5%9E%8B%E9%87%8F%E5%8C%96%E6%8E%A5%E5%8F%A3%2FPyTorch%2FCalibrator.md)
- [AntiOutlier.md](..%2FPython-API%E6%8E%A5%E5%8F%A3%E8%AF%B4%E6%98%8E%2F%E5%A4%A7%E6%A8%A1%E5%9E%8B%E5%8E%8B%E7%BC%A9%E6%8E%A5%E5%8F%A3%2F%E5%A4%A7%E6%A8%A1%E5%9E%8B%E9%87%8F%E5%8C%96%E6%8E%A5%E5%8F%A3%2FAntiOutlier.md)

#### modelslim_v1 服务配置协议

modelslim_v1是量化工具推出的新一代量化处理框架，目前正在快速演进中。

相较于v0版本，v1有以下优势：

- 算法独立实现，配置自由组合。
- 支持逐层量化，大幅降低资源消耗。
- 不依赖特定版本的CANN。

## modelslim_v1 量化服务配置详解

### 详细配置字段说明

V1量化服务的具体配置应当位于spec字段下。

#### pipeline - 量化流水线类型

**作用**: 定义量化处理的流水线类型。
**类型**: `string`。
**默认值**: `"auto"`。

| 可选值 | 说明 | 适用场景 | 特点 |
|--------|------|----------|------|
| auto | 工具自动选择 | 大多数场景 | 根据模型大小和可用内存自动选择最优策略 |
| layer_wise | 逐层量化 | 大模型（≥7B） | 内存占用低，可能需要适配 |
| model_wise | 非逐层量化 | 小模型（<7B） | 内存占用较高，兼容性好 |

#### dataset - 校准数据集配置

**作用**: 指定校准数据集文件名，将会从lab_calib目录下匹配使用的校准集。
**类型**: `string`。
**默认值**: `"mix_calib.jsonl"`。

| 属性 | 说明 |
|------|------|
| 文件位置 | lab_calib目录下 |
| 文件格式 | JSONL格式 |
| 用途 | 用于激活值量化的校准过程 |

#### 处理器配置字段 (process)

**作用**: 定义量化处理的处理器列表，按顺序执行每个处理器。

**特点**:

- **列表结构**: process 是一个处理器列表，包含多个处理器配置，不同的处理器配置以type字段为区分。
- **顺序执行**: 处理器按照在列表中的顺序依次执行。
- **灵活组合**: 可以组合不同类型的处理器实现复杂的量化策略。

##### 支持处理器表

###### linear_quant

**作用**: 对线性层进行量化处理。

**详细文档**: [linear_quant.md](features%2Flinear_quant.md)

**配置示例**:

```yaml
spec:
  process:
    - type: "linear_quant"
      qconfig:
        act:
          scope: "per_tensor"   # 激活值量化范围
          dtype: "int8"        # 激活值量化数据类型
          symmetric: false      # 是否对称量化
          method: "minmax"      # 量化方法
        weight:
          scope: "per_channel"   # 权重量化范围
          dtype: "int8"        # 权重量化数据类型
          symmetric: true       # 是否对称量化
          method: "minmax"      # 量化方法
      include: [ "*" ]           # 包含的层模式
      exclude: [ "*down_proj*" ] # 排除的层模式
```

**字段说明**:

| 字段名 | 作用 | 说明 |
|--------|------|------|
| type | 处理器类型标识 | 固定值"linear_quant"，用于标识这是一个线性层量化处理器 |
| qconfig | 线性层量化配置 | 包含激活值和权重的量化参数配置 |
| include | 包含的层模式 | 支持通配符匹配，指定要量化的层 |
| exclude | 排除的层模式 | 支持通配符匹配，优先级高于include |

###### group

**作用**: 对多个处理器进行分组管理，支持对同一组内的处理器进行统一的配置管理。

**配置示例**:

```yaml
spec:
  process:
    - type: "group"
      configs:
        - type: "linear_quant"
          qconfig: *default_w8a8
          include: ["*self_attn*"]
        - type: "linear_quant"
          qconfig: *default_w8a8_dynamic
          include: ["*mlp*"]
          exclude: ["*gate"]
```

**字段说明**:

| 字段名 | 作用 | 说明 |
|--------|------|------|
| type | 处理器类型标识 | 固定值"group"，用于标识这是一个分组处理器 |
| configs | 组内处理器配置列表 | 包含多个处理器配置，支持不同类型处理器组合 |

**使用场景**:

- 当需要对不同类型的层应用不同的量化策略时，可以使用group进行分组管理，可以降低资源消耗。
- 例如：对self_attention层使用静态量化，对mlp层使用动态量化。

#### 保存器配置字段 (save)

**作用**: 定义量化结果的保存器列表。

##### 支持保存器表

###### ascendv1_saver

**作用**: 保存为ascendv1格式。

**配置示例**:

```yaml
spec:
  save:
    - type: "ascendv1_saver"
      part_file_size: 4            # 分片文件大小（GB）
```

**字段说明**:

| 字段名 | 作用 | 说明 |
|--------|------|------|
| type | 保存器类型标识 | 固定值"ascendv1_saver"，用于标识这是一个Ascend格式保存器 |
| part_file_size | 分片文件大小 | 分片文件的大小，单位为GB |
