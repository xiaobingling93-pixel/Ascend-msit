# 使用背景
在量化大模型的时候，对于显存受限的设备，模型无法全部加载到显存中，可以启用低显存量化模式。

# 使用方法
启用低显存量化，主要涉及模型加载，异常值抑制配置和量化配置三个部分。

## 模型加载参数修改
使用transformer库的from_pretrained方法加载模型时，通过调整device_map和max_memory控制模型加载时的内存约束。device_map设置为auto，建议每张NPU卡的显存最大值设置为卡容量的80%，因为会有一些额外操作导致使用的显存增加；cpu内存最大值配置方式：cpu总内存 * 80% - 浮点模型的权重大小。
约束内存使用，使用示例如下：
```python
model = AutoModelForCausalLM.from_pretrained(
        pretrained_model_name_or_path=model_path,
        torch_dtype='auto',
        device_map="auto",
        max_memory={
            0:"25GiB",  #加载模型，NPU0最多使用25GB显存
            1:"25GiB",  #加载模型，NPU1最多使用25GB显存
            2:"25GiB",
            3:"25GiB",
            4:"25GiB",
            5:"25GiB"，
            "cpu": "500GiB"   #加载模型时，最多使用500GB的Host侧内存    
        }  
)
```


# 异常值抑制参数修改
要启用低显存量化，需要在AntiOutlierConfig中指定参数low_memory。
low_memory参数接受一个字典，目前支持字段memory，表示使用内存。将如下字段添加到AntiOutlierConfig当中即可：low_memory={'offload_type':'memory'}
使用示例如下：
```python 
anti_config = AntiOutlierConfig(w_bit=8, 
                                a_bit=8,
                                anti_method="m3", 
                                dev_type="npu", 
                                dev_id=model.device.index，
                                low_memory={'offload_type':'memory'}
                                )
```

# 量化参数修改
要启用低显存，还需要在QuantConfig中指定参数low_memory，将如下字段添加到QuantConfig当中即可：low_memory={'offload_type':'memory'}
