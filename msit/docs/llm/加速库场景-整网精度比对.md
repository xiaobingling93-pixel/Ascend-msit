# 整网精度比对

## 1. 加速库算子与 torch 模型算子的自动映射比对

### 自动对比思路

使用方法分为四步：

1. dump torch 模型算子 tensor 及拓扑信息
2. dump 加速库算子 tensor 及拓扑信息
3. 手动添加自定义算子映射（若不指定则使用内置算子映射）
4. 将 dump 目录及算子类型映射字典文件的目录作为入参输入到 msit llm compare 中完成比对

### 第一步：dump torch 模型算子 tensor 及拓扑信息

#### 使用方式
落盘torch模型算子 tensor 及拓扑信息时，需要在模型脚本中添加DumpConfig和register_hook接口及相关代码。

模型脚本示例（llama2 7b）：

```python
from transformers import AutoTokenizer, LlamaForCausalLM

tokenizer = AutoTokenizer.from_pretrained("/home/data/Llama-2-7b-hf", trust_remote_code=True)
model = LlamaForCausalLM.from_pretrained("/home/data/Llama-2-7b-hf", trust_remote_code=True) # 初始化模型实例model

# dump相关代码，在模型初始化后添加
from msit_llm import DumpConfig, register_hook
dump_config = DumpConfig(dump_path="./")
register_hook(model, dump_config)

# 继续执行推理脚本
prompt = "What's deep learning"
inputs = tokenizer(prompt, return_tensors="pt")
outputs = model.generate(inputs['input_ids'])
generated_text = tokenizer.decode(outputs[0], skip_special_tokens=True)
print(generated_text)
```

> DumpConfig 支持指定 dump 路径，还可以减小 dump 数据范围，包括指定需要 dump 的 token 列表，要 hook 的 module 类型，指定 dump 算子输入还是输出，等。详见[《在线推理 DUMP 功能使用说明》](/msit/docs/llm/工具-DUMP在线推理数据使用说明.md#dumpconfig)

> dump 数据路径：
>
> - dump 默认落盘路径`{DUMP_DIR}`在当前目录下，可以通过配置`dump_path`指定落盘路径。
> - tensor 信息落盘位置：在`{DUMP_DIR}/ait_dump_{TIMESTAMP}/torch_tensors/{device_id}_{PID}/{TID}/`目录下
> - model 拓扑信息落盘位置：生成在`{DUMP_DIR}/ait_dump_{TIMESTAMP}/torch_tensors/{device_id}_{PID}/model_tree.json`文件中
> - 注：`device_id`为设备号；`PID`为进程号；`TID`为`token_id`

### 第二步：dump 加速库算子 tensor 及拓扑信息

#### 使用方式

`msit llm dump --exec "<任意包含ATB的程序执行命令>" --type tensor model -er 1,1`

> - `--exec`是用于指定拉起执行加速库大模型推理脚本命令的参数
> - `--type`是用于指定dump类型的参数，`--type tensor model`指定 dump 模型的 tensor 及 model 拓扑信息
> - `--execute-range/-er`是用于指定dump的token轮次范围的参数，`-er 1,1`指定 dump 第 1 个 token 轮次
>   - 在 mindie RC1 版本之后，运行 atb 模型一般会先进行 warmup。导致第 0 个 token 轮次是 warmup 的数据，可以通过 er 参数指定 dump 第一个轮次解决该问题
>   - 在 mindie RC1 版本之后，atb 模型一般会有 prefill 和 decode 两个模型。prefill 只运行一次，所以在目录 1 中，会存在一次 prefill 推理 dump 数据和 decode 推理 dump 数据，会存储 2 个 token 的数据。后面编号的目录仅会存在 decode 一个 token 的 dump 数据
> - 更多参数，可以参考[加速库 DUMP 功能使用说明](/msit/docs/llm/工具-DUMP加速库数据使用说明.md)

> - dump 默认落盘路径`{DUMP_DIR}`在当前目录下，如果使用`--output/-o`指定目录，落盘路径则为指定的`{OUTPUT_DIR}`。
> - tensor 信息落盘位置：在`{DUMP_DIR}/ait_dump_{TIMESTAMP}/tensors/{device_id}_{PID}/{TID}/`目录下
> - model 拓扑信息落盘位置：在`{DUMP_DIR}/ait_dump_{TIMESTAMP}/model/{PID}/`目录下。由于 model 由 layer 组合而成，因此使用 model 时，默认同时会落盘 layer 信息。

### 第三步：手动添加自定义算子映射（可选）

#### 使用方式

新建算子类型映射字典文件`op_mapping_file.json`

文件内容示例：

```
{
    "LayerNormOperation": "LayerNorm",
    "LinearOperation": "Linear",
    "CommonLayer_outtensor0": ["GLMBlock_output_0", "BloomBlock_output_0"],
    "MlpGateLayerV2":["BloomMLP", "MLP"],
    "RmsNormOperation":["RMSNorm"],
    "SelfAttentionOperation":["CoreAttention"]
}
```

- 字典键为 atb 算子类型；值为对应的 torch 算子类型，可为 string 或 list 类型
- 若 tensor 数不唯一，可在下划线后添加 tensor 文件名指定 tensor 映射关系
- 用户指定的算子映射会在内置算子映射的基础上更新，默认使用内置算子映射

#### 内置算子映射

| 加速库算子类型         | torch 算子类型                                                    |
| ---------------------- | ----------------------------------------------------------------- |
| LayerNormOperation     | LayerNorm                                                         |
| LinearOperation        | Linear                                                            |
| CommonLayer_outtensor0 | GLMBlock_output_0 (ChatGLM2 6b)<br>BloomBlock_output_0 (Bloom 7b) |
| MlpGateLayerV2         | MLP (ChatGLM2 6b)<br>BloomMLP (Bloom 7b)                          |
| RmsNormOperation       | RMSNorm                                                           |
| SelfAttentionOperation | CoreAttention (ChatGLM2 6b)                                       |

(扩充中...）

### 第四步：将 dump 目录及算子类型映射字典文件的目录作为入参输入到 msit llm compare 中完成比对

#### 使用方式

```shell
msit llm compare -gp {DUMP_DIR}/ait_dump_{TIMESTAMP}/torch_tensors/{device_id}_{PID}/{TID}/ -mp {DUMP_DIR}/ait_dump_{TIMESTAMP}/tensors/{device_id}_{PID}/{TID}/ --op-mapping-file xx/xxx/xx/
```

msit llm compare 提供有精度问题的数据与标杆数据之间的比对能力。

> - `--golden-path/-gp` 是用于指定标杆数据路径的参数，示例中为第一步中 `torch_tensor`所在目录 `{DUMP_DIR}/ait_dump_{TIMESTAMP}/torch_tensors/{device_id}_{PID}/{TID}/`
> - `--my-path/-mp` 是用于指定带比较的数据路径的参数，示例中为第二步中 `atb_tensor`所在目录 `{DUMP_DIR}/ait_dump_{TIMESTAMP}/tensors/{device_id}_{PID}/{TID}/`
> - `--op-mapping-file/-mf` 是用于指定算子类型映射关系文件目录路径的参数，默认为当前路径。示例中为第三步中算子类型映射字典文件`op_mapping_file.json`所在目录，若目录下无法找到`op_mapping_file.json`文件则使用内置算子映射
> - 完成比对后会生成一个 `msit_cmp_report_{TIMESTAMP}.csv`，保存比对的最终结果。默认文件保存路径在当前目录下，如果使用`--output/-o`指定目录，保存路径则为指定的`{OUTPUT_DIR}`。
> - csv 报告查看参考[精度比对结果参数说明](/msit/docs/llm/精度比对结果参数说明.md)
