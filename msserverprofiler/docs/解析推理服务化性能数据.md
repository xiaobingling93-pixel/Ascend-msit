# 解析推理服务化性能数据

该工具支持将从MindIE Server采集的性能数据解析。

## 1 数据准备

准备已经使用msprof进行初步解析的性能数据，确认存在数据文件`${your_file}/PROF_XXXX/host/sqlite/msproftx.db`，并且数据库中存在名为`MsprofTxEx`的表。

确认存在CPU数据文件`${your_file}/PROF_XXXX/host/sqlite/host_cpu_usage.db`，并且数据库中存在名为`CpuUsage`的表。

确认`${your_file}/PROF_XXXX`路径下存在配置文件`host_start.log`和`info.json`。

如果是刚刚采集的数据。建议使用 `msprof --export=on --output=PROF_XXXX` 先预处理一下

## 2 环境依赖

Python 3.7及以上

Pandas 2.x

## 3 执行解析脚本
进入`ms_server_profiler_analyze`文件夹，执行`analyze.py`脚本，具体命令如下：
```commandline
python3 analyze.py --input_path ${your_file} --output_path ${output_dir} --exporter ${exporter_list}
```
### 命令行参数

| 参数名                           | 描述                                        | 必选 |
|-------------------------------|-------------------------------------------| ---- |
| --input_path                       | 指定数据所在文件夹，会遍历读取该文件夹下所有名为`msproftx.db`的数据库 | 是   |
| --output_path                      | 指定解析后json文件生成路径，不传参默认在当前文件夹下生成            | 否   |
| --exporter_list                      | 指定需要解析的指标            | 否   |

目前支持的exporter有：
| exporter                           | 描述                                        | 结果文件 |
|-------------------------------|-------------------------------------------| ---------------|
|  trace                      | 服务化推理的trace图 | `chrome_tracing_${timestamp}.json` |
|  req_status                      | 服务化推理的请求状态图            | `request_status.jpg` (后续支持Prometheus格式)|


## 4 可视化
打开`chrome://tracing`网页，点击`load`将`chrome_tracing_${timestamp}.json`上传，即可查看可视化结果。
