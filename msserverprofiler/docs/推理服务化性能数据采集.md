# 推理服务化性能数据采集

该工具支持服务化数据采集。

## 1 采集接口

1. 头文件, 所有对外的接口都在 msSererProfiler.h , 均为开源代码。具体接口可以直接访问
    ```cpp
    #include "msSererProfiler.h"
    using namespace msServerProfiler;

    ```
2. 过程类数据采集，记录一个过程的开始和结束时间
    
    ```cpp
    auto prof = Span<Level::INFO>("span_name", resId); // resid 为当前资源序号，比如一个请求的编号
    prof.Start(); // 开始
    
    xxxx
    
    prof.End(); // 结束
    ```
3. 数值类数据采集，记录一个数值的变化，比如队列长度，池子大小，资源利用率
    ```cpp
    Metric<>::AddMetric("metric_name_cpu_usage", resId, 0.95);
    ```
4. 事件类数据采集，记录一个独立事件，比如告警，内存使用超过阈值
    ```cpp
    Event<>::AddEvent("event_name_warning", resId);
    ```
5. 资源关联类采集，同一个请求在不同系统可能表现为不同的资源id。关联不同的资源id ，可以将多个子系统关联起来

    ```cpp
    ResLink<>::AddLink(resId1, resId2);
    ```
6. 添加属性，上述接口都可以添加属性

    ```cpp
    auto prof = Span<Level::INFO>("span_name", resId);
    prof.AddAttr("attr_name", 123);

    auto metric = Metric<>("metric_name_cpu_usage", resId);
    metric.AddAttr("attr_name", "attr_str_value");
    metric.Mark(88);
    ```
7. python 实现了一模一样的接口。引入脚本如下：
    ```python 
    from ms_server_profiler import Span
    with Span("span_name", res_id, 20):
        xxx
    ```

## 2 编译

1. 当前库编译
```shell
mkdir -p build
cd build
cmake ..
make
cmake --install . --prefix ../../../server_profiler # 安装目录为 ../../../server_profiler
```

2. c++ 服务化编译
    1. 将 安装 目录的 include 目录 放到 CMakeList.txt 的 include_directories 中
    2. 将 安装 目录的 lib 目录 放到 CMakeList.txt 的 link_directories 中  
    3. 将 ms_server_profiler 放到要编译的输出件的 TARGET_LINK_LIBRARIES 中
    4. enjoy

## 使用

安全目录为 $INSTALL_PATH

```shell
source $INSTALL_PATH/scripts/set_env.sh
```
